# SaaS UX Patterns

UX best practices for SaaS landing pages, dashboards, onboarding flows, and conversion optimization.

## Landing Page Structure

Optimal order for SaaS landing pages:

1. **Hero** — Value proposition + primary CTA (above the fold)
2. **Social proof** — Logos, metrics, or testimonials
3. **Features** — 3-6 key features with visuals
4. **How it works** — 3-step process
5. **Pricing** — Clear tiers with recommended plan highlighted
6. **Testimonials** — Detailed customer stories
7. **FAQ** — Address objections
8. **Final CTA** — Repeat primary call to action

## Key Principles

- **Above the fold:** Hero + CTA visible without scrolling
- **Time to value:** Minimize steps from signup to first meaningful action
- **Progressive disclosure:** Don't overwhelm — reveal complexity gradually
- **Mobile first:** Design for mobile, then enhance for desktop
- **F-pattern reading:** Place key info along F-pattern for LTR layouts
